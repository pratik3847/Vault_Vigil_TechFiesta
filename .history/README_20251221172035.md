# Guardian AI - Fraud Detection & Risk Management System

An enterprise-grade AI-powered fraud detection platform built with React, TypeScript, and Vite. This system demonstrates a complete fraud prevention pipeline from transaction simulation to automated decision-making.

## 🎯 Project Overview

Guardian AI is a comprehensive fraud detection system designed for fintech applications. It combines transaction monitoring, network analysis, risk scoring, and automated decision-making to prevent fraudulent activities in real-time.

### System Architecture

```
Transaction Input → Link Analysis → Risk Scoring → Decision Engine
```

1. **Transaction Simulator** - Generate and test transaction patterns
2. **Fraud Link Analysis** - Detect suspicious account networks
3. **Risk Scoring Engine** - Multi-factor risk assessment
4. **Decision Engine** - Automated fraud decisions with transparent reasoning

## 🚀 Features

### 1. Transaction Simulator
- Manual transaction input with full details (accounts, amount, location, device, IP)
- Auto-generate normal transactions (realistic patterns)
- Auto-generate fraudulent transactions (anomaly simulation)
- Real-time risk scoring and status display
- Transaction history table with sorting and filtering

**Fraud Detection Logic:**
- High-value transactions (>$10,000)
- Suspicious locations (unknown, high-risk countries)
- Transaction velocity patterns
- Device/IP reuse detection

### 2. Fraud Link Analysis (INPUT-DRIVEN)
**CRITICAL: This module accepts user-defined links, not just displays hardcoded data**

**Input Methods:**
- Manual form input (Account A → Account B relationships)
- Link types: Transaction, Shared Device, Shared IP
- Bulk JSON import for multiple links

**Analysis Features:**
- Shared device detection (multiple accounts → same device = HIGH RISK)
- Shared IP detection (multiple accounts → same IP = HIGH RISK)
- Circular money flow detection (A→B→C→A patterns)
- Fraud chain identification
- Link risk scoring (0-100)

**Output:**
- Interactive network graph visualization
- Highlighted risky links (color-coded by risk level)
- Detected fraud chains with details
- Circular pattern alerts

### 3. Risk Scoring Engine
- Search by Transaction ID or Account ID
- Multi-factor risk analysis:
  - Transaction anomalies
  - Network/link risk
  - Device/IP reuse
  - Location mismatches
  - Historical patterns

**Risk Score Breakdown:**
- 0-30: Safe (Green)
- 31-70: Suspicious (Yellow)
- 71-100: Fraud (Red)

**Contributing Factors Display:**
- Linked fraud accounts count
- Abnormal amount flags
- Location mismatches
- Device/IP reuse indicators

### 4. Decision Engine
**Final Stage: Automated Decision Making**

**Decision Logic:**
- **ALLOW** (Risk 0-30): Process normally
- **ALERT** (Risk 31-70): Require 2FA/review
- **BLOCK** (Risk 71-100): Immediate blocking

**Risk Components (Weighted):**
- Transaction Risk: 40% weight
- Network Risk: 30% weight
- Velocity Risk: 20% weight
- Location Risk: 10% weight

**Output:**
- Clear decision: ALLOW / ALERT / BLOCK
- Detailed reasoning (why this decision)
- Actionable recommendations
- Risk score breakdown visualization

## 🚀 Getting Started

### Installation

```sh
# Clone the repository
git clone <YOUR_GIT_URL>

# Navigate to project directory
cd guardian-ai-dashboard-main

# Install dependencies
npm install
# or
bun install

# Start development server
npm run dev
# or
bun dev
```

The application will be available at `http://localhost:8080`

### Build for Production

```sh
npm run build
# or
bun build

# Preview production build
npm run preview
```

```typescript

  transactionRisk * 0.4 +
  networkRisk * 0.3 +
  velocityRisk * 0.2 +
  locationRisk * 0.1
```

### Link Risk Scoring
```typescript

  (sharedDevices * 15) +
  (sharedIPs * 20) +
  (highVelocity ? 25 : 0)
```
